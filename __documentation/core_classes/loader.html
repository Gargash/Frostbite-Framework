Loader Class

The loader does what its name says... It loads stuff. The loader class makes it easy to load libraries, modules, helpers, and Database connections and instance them in the Main controller.

Loading the Loader Class

There will be times when you will need to manually load the loader class. This is explained below.

Note:  This class always loaded before the main controller is initialized, so you will not need to initialize it by default.

$this->load = load_class('Loader');

Using the loader file to load a library

When loading a library, it will always load the custom library file from the Application folder OVER the Core one. If you are merely just wanting to add to a core library, you will need to extend it using parent::__contruct() in your app library class constructer.

$this->load->library('$class_name');

The above example shows the minimum to load a library. There is 1 more argument that can be passed as shown below, but are not required:

$this->load->library('$class_name', $instance');
Arguments:

  1. $classname [Type: String]: The library class name.
  2. $instance [Type: Bool, Default TRUE]: Do we instance the class in the controller?.

Using the loader to establish a Database Connection

Before you establish a database connection, you need to enter the connection info in the "Application/config/database.config.php" file. Please see Database Class for more details. Below is an example of how to configure you DB connection: 

		$DB_configs = array( 
			'$config_key' => array(
				'hostname' => ...

Now below, this how we load that very connection: $connection = $this->load->database('$config_key', '$instance');

Arguments:

  1. $instance [Type: Bool, Default: FALSE]: Do we instance this connection in the controller super object? NOTE: If $instance is set to TRUE, then $connection = is NOT needed as it will be instanced as "$this->$config_key" UNLESS you are using it outside the controller.

Using the loader to load a Model

When loading a model, its important to know that any and all libraries and DB connections you have loaded will now be available UNLESS you use the get_instance(); function.
$name is the name of the model class we are loading.

$this->load->model('$name', '$instance_as');
$this->$name->method();
Arguments:

  1. $instance_as [Type: String, Default: Lowercase $name]: Setting this will allow you to access the model using a different name then the class name

$this->$instance_as->method();

Using the loader to load Helper files

Helper files are just files that contain a list of functions for you to use, so all you have to do is load the helper, and the functions are globally available for use.

$this->load->helper('$name');
Arguments:

  1. $name [Type: String]: The name of the helper file, No extension
