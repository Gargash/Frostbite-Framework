Views

A view is basically a webpage full of html code and things of that sort. You can embed views inside of views if you wish. Views are never accessed directly via the url as the controller is responsible for loading the view.

Creating a View

Using the example controller in the controller page, we are going to make a view for it. 

<html>
<head>
	<title>My View File</title>
</head>
<body>
	<div>Welcome to my view file!</div>
</body>
</html>
  1. First, Create this file: "application/views/test_page.php"
  2. Next, copy the code above and put it in your newly created file
  3. Keep reading to find out how to load this view

Loading a View

to load the view file we just created, you would do it like so: 

public function index()
{
	$this->load->view('test_page'); 
}
You can load multiple views at once if you wish like so: 
public function index()
{
	$this->load->view('header'); 
	$this->load->view('body');
	$this->load->view('footer');
}
Your view files can be stored in subfolders just like Models of you wish. They are called like this: $this->load->view('folder/filename');

Passing variables into the view file

Variables can be passed into the view by using an array. Look at the example below 

public function index()
{
	$data['title'] = 'Testpage';
	$data['body'] = 'It Works!';
	
	$this->load->view('test_page', $data);
}
And here is what the view looks like: 
<html>
<head>
	<title><?php echo $title; ?></title>
</head>
<body>
	<div><?php echo $body; ?></div>
</body>
</html>
When the view is loaded into the browser, the php variables will be processed and replaced. You can do anything you want using php in these view files. You can do loops, objects, you name it.

Returning the view as a String

There is a third variable that can be passed into the loader that tells it to return the page as a string, instead of sending it to the browser. $this->load->view('index', '', TRUE); By default, this third param is set to FALSE and the page is passed to the browser for display.
