Session Class

The session class makes it easier to manage users login session information, and is more secure as it does not use the $_SESSION global. All session information is stored in the database for easy and secure reach. If you have the session Database turned off, then session data will be serialized and stored in a file.

Initializing the Session Class

By default, the session class is not initiated, so you will need to set that up manually or enable in the config to autoload the class. Once loaded, the session class will set itself up automatically, so there isnt much you need to do. All you have to do is use the loader to load the session like so:

$this->load->library('Session');

Public class variables

There are a few variables within the Session Class that can be accessed from the outside

$data

The data variable is an array of variables you have set using the "set" method. (Type: Array)

Format: $data = array( 'key' => 'value', 'key2' => 'value2' ... etc etc);
Access: $session_data = $this->session->data;

Setting a session variable

Once the session is loaded, you will most likely want and need to set some session variables, such as a username etc etc. Here is how you do that:

$this->session->set('$key', '$value');
Arguments:

  1. $key [Type: String]: This is the name of the value your setting, such as "username = $value", the $key in this example is "username".
  2. $value [Type: Mixed]: This is value you are setting to your key.

Fetching a session variable

Once you have set some session variables, you will probablily need to access one of those variables later on. This is how you do that:

$var = $this->session->get('$key');
Arguments:

  1. $key [Type: String]: This is the name of the value your getting.

Saving / Updating your Session

Once you have all your session data set up, you will need to save it to the database and cookie. If session data already exists, then it will update where needed. This is how you save your session once all the data is set.

$this->session->save();
