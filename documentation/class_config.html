Config Class

The config class handles 4 different config files. The site config (congig.php), The core config (core.config), The database config (database.coonfig.php), and the module config if A) We are loding a module and B) The config file exists.

Note:  This class should never handled directly. Use the config functions to get, set, and save config variables.

Features:

  • Allows saving of site config (config.php) variables via an Admin panel etc etc. Note that doing this will remove any and all comment from the file.

Loading the Config class

If you need to access the function manually, you will do so like this

$this->load->library('Config');

Getting a variable from the Site config file

$this->config->get('$var_name');

Getting a variable from the Core or Database config file

$this->config->get('$var_name', 'Core');
OR
$this->config->get('$var_name', 'DB');

Setting a variable (Site config only!)

$this->config->set('$var_name', '$new_value');

Saving all set config variables to the config.php (Site config only!)

$this->config->Save();
